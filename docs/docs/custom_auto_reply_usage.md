# 自定义定时回复功能使用说明

## 功能概述

自定义定时回复功能允许你创建多个定时回复任务，可以向任意群组或用户发送定时消息。相比 OurBits 定时回复，这个功能更加灵活和通用。

## 主要特性

- ✅ 支持多个独立的定时回复任务
- ✅ 可自定义目标聊天（群组或私聊）
- ✅ 可选择使用用户账号或 Bot 账号发送
- ✅ 灵活的定时配置（支持多个时间点）
- ✅ 可选的活动时间范围限制
- ✅ 完整的任务管理（启用/关闭/删除）

## 使用方法

### 1. 进入管理界面

发送命令：
```
/customreply
```
或
```
.customreply
```

### 2. 创建新任务

使用命令：
```
/customreply_add <任务ID>
```

例如：
```
/customreply_add task1
```

任务ID 支持中文、字母、数字、下划线等，用于唯一标识一个任务（最多50个字符）。

### 3. 配置任务

创建任务后，在管理界面点击任务名称进入详情页面，可以配置以下内容：

#### 3.1 任务名称
- 点击 **✏️ 修改任务名称**
- 回复新的任务名称（用于显示，可以是中文）

#### 3.2 目标聊天ID
- 点击 **🎯 设置目标聊天**
- 回复目标聊天的 ID
  - 群组ID：通常是负数，如 `-1001234567890`
  - 用户ID：通常是正数，如 `123456789`

**如何获取聊天ID？**
- 群组：将 Bot 添加到群组，在群组发消息，查看日志
- 用户：查看用户的 Telegram ID

#### 3.3 回复内容
- 点击 **📝 设置回复内容**
- 回复要发送的消息内容
- 支持 Markdown 格式
- 最多 4096 个字符

#### 3.4 定时配置
- 点击 **⏰ 设置定时配置**
- 回复定时配置，格式：`小时,分钟`

**示例：**
```
0,3,6,9,12,15,18,21,0
```
表示每天 0、3、6、9、12、15、18、21 点的 0 分执行

```
12,30
```
表示每天 12 点 30 分执行

```
0,6,12,18,0
```
表示每天 0、6、12、18 点的 0 分执行

#### 3.5 活动时间范围（可选）
- 点击 **📅 设置活动时间**
- 回复时间范围，格式：`开始时间|结束时间`

**示例：**
```
2025-11-21 00:00:00|2025-11-23 21:04:00
```

如果不需要时间限制，直接回复空消息即可。

### 4. 启用任务

配置完成后：
1. 点击 **🟢 启用任务**
2. 任务将按照设定的时间自动执行

### 5. 管理任务

在任务详情页面可以：
- **🔴 关闭任务** / **🟢 启用任务**：切换任务状态
- **🗑️ 删除任务**：永久删除任务
- **🔙 返回任务列表**：返回主界面

## 使用场景

### 场景 1：群组定时公告
```
任务名称：每日公告
目标聊天：-1001234567890（你的群组ID）
定时配置：9,0（每天9点）
回复内容：
📢 早上好！今天是美好的一天！
记得签到领取奖励哦~
```

### 场景 2：多时段提醒
```
任务名称：活动提醒
目标聊天：-1001234567890
定时配置：0,6,12,18,0（每6小时一次）
活动时间：2025-12-01 00:00:00|2025-12-31 23:59:59
回复内容：
🎉 活动进行中！快来参加吧！
```

### 场景 3：私聊定时消息
```
任务名称：每日问候
目标聊天：123456789（某个用户的ID）
定时配置：8,30（每天8:30）
回复内容：
早安！今天也要加油哦！💪
```

## 注意事项

1. **权限要求**
   - 你的用户账号必须在目标群组中
   - 如果是私聊，需要对方允许你发送消息

2. **时间格式**
   - 定时配置使用 24 小时制
   - 活动时间必须严格按照 `YYYY-MM-DD HH:MM:SS` 格式

3. **任务ID**
   - 任务ID 创建后不能修改
   - 支持中文、字母、数字等，最多50个字符
   - 建议使用有意义的名称，如 `每日签到`、`daily_notice`、`event_reminder` 等

4. **消息发送**
   - 消息发送成功后会在 Bot 消息聊天中收到通知
   - 如果发送失败，也会收到错误通知

5. **任务数量**
   - 理论上可以创建无限个任务
   - 建议根据实际需求合理创建，避免过多任务

## 常见问题

### Q: 如何测试任务是否配置正确？
A: 可以先设置一个较近的时间点（如当前时间后5分钟），启用任务后观察是否正常发送。

### Q: 任务没有按时执行怎么办？
A: 
1. 检查任务是否已启用（绿色勾）
2. 检查定时配置是否正确
3. 如果设置了活动时间，检查当前时间是否在范围内
4. 查看日志文件确认是否有错误

### Q: 可以同时向多个群组发送吗？
A: 需要为每个群组创建单独的任务。

### Q: 如何暂停任务？
A: 点击任务详情中的 **🔴 关闭任务** 按钮即可暂停，需要时再启用。

### Q: 删除任务后可以恢复吗？
A: 不可以，删除是永久性的。建议先关闭任务而不是删除。

## 与 OurBits 定时回复的区别

| 特性 | OurBits 定时回复 | 自定义定时回复 |
|------|-----------------|---------------|
| 任务数量 | 1个固定任务 | 无限个任务 |
| 目标聊天 | 固定 OurBits 群组 | 任意群组或用户 |
| 定时配置 | 固定时间点 | 完全自定义 |
| 发送者 | 固定用户账号 | 可选用户或Bot |
| 配置方式 | 菜单配置 | 菜单+命令配置 |

## 技术说明

- 配置存储在 `config/state.toml` 文件中
- 使用 APScheduler 进行任务调度
- 支持 Cron 表达式定时
- 任务状态实时同步
