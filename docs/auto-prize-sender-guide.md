# 手动发奖功能使用指南

## 功能简介

手动发奖功能可以记录你发起的抽奖开奖信息，然后通过命令手动发奖。发奖方式是回复中奖者的参与消息，发送 `+奖品数量` 格式的消息。

## 功能特点

- ✅ 自动记录你发起的抽奖开奖信息
- ✅ 手动触发发奖（输入命令）
- ✅ 支持批量发奖（一次发送多个抽奖）
- ✅ 支持多种奖品数量格式（1w、5000、100魔力等）
- ✅ 模拟人工操作（随机等待时间）
- ✅ 详细的发奖通知
- ✅ 完善的错误处理

## 使用方法

### 1. 开启发奖记录功能

1. 打开 bot 菜单
2. 进入"自动抽奖设置"
3. 找到"发奖记录功能"
4. 点击切换为"开"

### 2. 发起抽奖

正常使用抽奖机器人发起抽奖即可。

### 3. 查看待发奖列表

当抽奖开奖后，系统会自动记录并发送通知：

```
📝 记录待发奖抽奖

抽奖ID: 259f40cc-cce6-46e7-9665-c12c12a35f20
中奖人数: 3
群组: 观众

输入命令发奖: /sendprize 259f40cc
查看待发奖列表: /listprize
```

使用命令查看所有待发奖抽奖：

```
/listprize
```

或者使用带点号的格式：

```
.listprize
```

**注意：** 可以在任何群组或私聊中使用命令，只要是你自己发送的。

输出示例：

```
📝 待发奖抽奖列表

1. ID: 259f40cc...
   中奖: 3人 | 群组: 观众
   时间: 02-14 13:56

2. ID: 3a8b9c2d...
   中奖: 5人 | 群组: SSD
   时间: 02-14 14:20

总计: 2 个待发奖抽奖

发奖命令: /sendprize [ID前缀] 或 .sendprize [ID前缀]
发送全部: /sendprize all 或 .sendprize all
清空列表: /clearprize 或 .clearprize
```

### 4. 手动发奖

使用命令发奖（支持 `/` 和 `.` 两种前缀）：

```bash
# 发送指定抽奖的奖品（使用ID前缀）
/sendprize 259f40cc
# 或
.sendprize 259f40cc

# 发送所有待发奖抽奖的奖品
/sendprize all
# 或
.sendprize all
```

**注意：** 可以在任何群组或私聊中使用命令。

系统会：
1. 找到匹配的抽奖
2. 提取参与消息链接
3. 回复参与消息发送 `+奖品数量`
4. 发送完成通知

### 5. 清空待发奖列表

如果不想发奖了，可以清空列表：

```
/clearprize
```

或者：

```
.clearprize
```

## 工作流程

### 1. 记录开奖信息

当抽奖开奖后，系统会：

1. **监听开奖消息**
   - 格式: "参与人数够啦！！开奖~"

2. **解析抽奖信息**
   - 抽奖ID
   - 创建者ID
   - 中奖者列表
   - 奖品名称和数量

3. **验证记录条件**
   - 确认是你发起的抽奖
   - 确认你自己没有中奖

4. **保存到待发奖列表**
   - 记录所有中奖者信息
   - 记录参与消息链接
   - 发送通知

### 2. 手动发奖

当你输入发奖命令后，系统会：

1. **查找匹配的抽奖**
   - 根据ID前缀匹配
   - 或发送所有待发奖抽奖

2. **提取参与消息链接**
   - 从开奖消息中提取每个中奖者的参与消息链接
   - 格式: `https://t.me/c/群组ID/消息ID`

3. **发送奖品**
   - 回复中奖者的参与消息
   - 发送格式: `+奖品数量`
   - 例如: `+10000`、`+5000`

4. **发送通知**
   - 每次发奖失败时，发送失败通知
   - 发奖完成后，发送总结通知

5. **清理列表**
   - 发奖完成后，从待发奖列表中移除

## 奖品数量识别

系统支持多种奖品数量格式：

| 奖品名称 | 识别结果 | 发送消息 |
|---------|---------|---------|
| 1w | 10000 | +10000 |
| 5000 | 5000 | +5000 |
| 100魔力 | 100 | +100 |
| 1k | 1000 | +1000 |
| 10000*10 | 10000 | +10000 |

注意：
- `1w` 表示 1万 = 10000
- `1k` 表示 1千 = 1000
- `10000*10` 提取第一个数字（单个奖品价值）

系统会：

1. 提取参与消息链接（从"参与消息"超链接）
2. 回复参与消息发送: `+10000`

## 时间控制

为了模拟人工操作，系统会：

- 开始发奖前等待 3-8 秒（随机）
- 每次发奖后等待 2-5 秒（随机）

## 使用场景

### 场景1: 单个抽奖发奖

1. 发起抽奖
2. 等待开奖
3. 收到记录通知
4. 输入 `/sendprize 259f40cc`
5. 完成发奖

### 场景2: 批量发奖

1. 发起多个抽奖
2. 等待所有抽奖开奖
3. 收到多个记录通知
4. 输入 `/listprize` 查看列表
5. 输入 `/sendprize all` 批量发奖
6. 完成所有发奖

### 场景3: 选择性发奖

1. 发起多个抽奖
2. 等待所有抽奖开奖
3. 输入 `/listprize` 查看列表
4. 选择要发奖的抽奖
5. 输入 `/sendprize [ID前缀]` 发送指定抽奖
6. 对不想发奖的抽奖，输入 `/clearprize` 清空

## 注意事项

1. **只记录你发起的抽奖**
   - 系统会检查创建者ID
   - 只有你发起的抽奖才会被记录

2. **不会记录自己中奖的抽奖**
   - 如果你自己中奖了，不会记录该抽奖
   - 避免自己给自己发奖的尴尬情况

3. **需要参与消息链接**
   - 系统从开奖消息中提取参与消息链接
   - 如果无法提取链接，会跳过该中奖者并发送通知

4. **奖品数量为0**
   - 如果无法识别奖品数量，会跳过发奖
   - 会在日志中记录警告

5. **待发奖列表存储在内存中**
   - 重启程序后，待发奖列表会清空
   - 建议及时发奖，不要积累太多

6. **ID前缀匹配**
   - 使用ID前缀匹配，不需要输入完整ID
   - 例如: `/sendprize 259f` 可以匹配 `259f40cc-cce6-46e7-9665-c12c12a35f20`

## 故障排查

### 问题1: 没有记录开奖信息

**可能原因：**
- 发奖记录功能未开启
- 不是你发起的抽奖
- 你自己中奖了

**解决方法：**
1. 检查发奖记录功能开关状态
2. 确认抽奖创建者是你
3. 查看日志了解跳过原因

### 问题2: 发奖失败

**可能原因：**
- 无法提取参与消息链接
- 无法识别奖品数量
- 网络问题

**解决方法：**
1. 查看bot消息群的失败通知
2. 检查日志中的错误信息
3. 手动发奖给失败的中奖者

### 问题3: 找不到待发奖抽奖

**可能原因：**
- ID前缀输入错误
- 待发奖列表已清空
- 程序重启导致列表丢失

**解决方法：**
1. 使用 `/listprize` 查看当前列表
2. 检查ID前缀是否正确
3. 如果列表为空，需要等待新的开奖

### 问题4: 奖品数量识别错误

**可能原因：**
- 奖品名称格式不标准
- 不支持的格式

**解决方法：**
1. 查看日志中的警告信息
2. 手动修正发奖数量
3. 反馈给开发者改进识别规则
